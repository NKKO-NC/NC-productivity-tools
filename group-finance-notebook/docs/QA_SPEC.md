# QA 測試規格

## 1. QA 目標

Phase 1 QA 的核心目標是確認：

- 工具可純本機使用。
- 公司識別穩定。
- 投資架構可依時間正確切換。
- 匯入匯出不破壞資料。
- UI 圖面操作符合財務人員工作情境。
- 歷史資料不會被修改洗掉。

## 2. 純本機測試

### 2.1 測試條件

在無網路環境下執行：

- 開啟工具。
- 開啟既有 DB。
- 新增公司。
- 新增投資關係。
- Excel 貼上匯入。
- 切換 as-of date。
- 匯出 DB。
- 匯出 CSV。

### 2.2 通過條件

- 所有功能可正常執行。
- 不要求登入。
- 不要求連線。
- 不呼叫外部 API。
- 不因無網路而阻擋核心功能。

## 3. 公司識別測試

### 3.1 測試資料

情境：

- 同公司同時有公司代碼、簡稱、全稱。
- 不同公司有相同簡稱。
- 公司曾改名。
- 公司缺少代碼但有全稱。
- 公司名稱含空白或全半形差異。

### 3.2 通過條件

- 公司代碼可找到同一 company_id。
- 公司簡稱可找到同一 company_id。
- 公司全稱可找到同一 company_id。
- 簡稱重複時不得自動亂配。
- 舊名稱仍可查到同一 company_id。
- unresolved 公司需在匯入 preview 顯示。

## 4. 投資架構時間測試

### 4.1 基準資料

```text
2026-03-31: A 公司持有 B 公司 60%
2026-06-30: A 公司持有 B 公司 80%
2026-06-30: C 公司新增持有 D 公司 100%
```

### 4.2 通過條件

- 選 2026-03-31 顯示 A -> B 60%。
- 選 2026-06-30 顯示 A -> B 80%。
- 選 2026-03-31 不顯示 C -> D。
- 選 2026-06-30 顯示 C -> D 100%。
- 修改 2026-06-30 資料時，不影響 2026-03-31 圖面。

## 5. 匯入測試

### 5.1 Excel 貼上測試

測試情境：

- 正常欄位順序。
- 欄位順序打亂。
- 中文欄名。
- 缺少非必要欄位。
- 缺少必要欄位。
- 百分比格式為 80%。
- 百分比格式為 0.8。
- 百分比格式為 80。
- 日期格式為 2026/3/31。
- 日期格式為 2026-03-31。

### 5.2 通過條件

- 欄位可重新 mapping。
- 錯誤資料不直接入庫。
- 匯入前有 preview。
- 匯入後產生 source_batch_id。
- 匯入失敗可回到修正。
- 既有資料不被破壞。

## 6. UI 圖面測試

### 6.1 功能測試

通過條件：

- 點公司卡片可開右側編輯。
- 點投資線可開右側編輯。
- 搜尋公司可聚焦節點。
- 可拖曳節點。
- 可放大縮小。
- 可平移畫布。
- 可自動排版。
- 可切換 as-of date。
- 資料缺漏時圖卡有提示。

### 6.2 布局記憶測試

通過條件：

- 拖曳公司後位置被保存。
- 切換日期後仍存在的公司沿用位置。
- 日期切回來後原布局仍在。
- 新增公司不會重排整張圖。

## 7. 效能測試

### 7.1 Phase 1 目標

```text
100 家公司 / 200 條投資線：流暢操作
500 家公司 / 1000 條投資線：可操作，允許較慢
```

### 7.2 通過條件

- 100 家公司載入時間在可接受範圍內。
- 100 家公司切換 as-of date 不明顯卡死。
- 500 家公司仍可搜尋、縮放、查看公司資料。

## 8. 匯出還原測試

### 8.1 測試流程

1. 建立測試資料。
2. 匯出 SQLite DB。
3. 匯出 CSV 檔案組。
4. 另開新工作資料夾。
5. 重新匯入 DB。
6. 重新匯入 CSV。
7. 比對 as-of date 查詢結果。

### 8.2 通過條件

- 公司數一致。
- alias 對應一致。
- metadata 版本一致。
- 投資架構版本一致。
- graph layout 可還原。
- as-of date 圖面一致。

## 9. 歷史不可覆蓋測試

### 9.1 測試流程

1. 建立 A -> B 60%，有效日 2026-03-31。
2. 修改為 A -> B 80%，有效日 2026-06-30。
3. 查詢 2026-03-31。
4. 查詢 2026-06-30。

### 9.2 通過條件

- 2026-03-31 仍為 60%。
- 2026-06-30 為 80%。
- DB 中保留兩筆版本紀錄。
- 舊版本 valid_to 正確關閉。

## 10. Phase 1 總驗收

同一份本機資料，在選取 2026-03-31 與 2026-06-30 時，可以正確顯示不同的公司投資架構，且所有修改都不覆蓋歷史資料。
