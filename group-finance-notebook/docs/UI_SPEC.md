# UI 規格

## 1. UI 原則

- 第一畫面就是可操作的工作台，不做行銷式首頁。
- 圖像檢視優先，表格和表單作為輔助。
- 財務人員需要密度高、可掃描、可驗證的 UI。
- 所有修改都必須能指定生效日期。
- 切換時間時，不得洗掉使用者調整過的圖面布局。

## 2. 主畫面布局

```text
┌────────────────────────────────────────────────────────────┐
│ 頂部工具列：檔案 / 匯入 / 匯出 / 搜尋 / 視圖模式              │
├──────────────┬──────────────────────────────┬──────────────┤
│ 左側面板      │ 中央投資架構圖                 │ 右側編輯面板  │
│ 資料總覽      │ 公司圖卡 + 投資連線             │ 公司/連線資料 │
│ 篩選器        │ 放大縮小 / 拖曳 / 自動排版       │ 版本歷史      │
│ 圖層          │                              │ 驗證訊息      │
├──────────────┴──────────────────────────────┴──────────────┤
│ 底部時間軸：as-of date / 匯入狀態 / 資料驗證摘要              │
└────────────────────────────────────────────────────────────┘
```

## 3. 頂部工具列

必要控制：

- 新增公司
- 新增投資關係
- 匯入
- 匯出
- 搜尋公司
- 自動排版
- 重置縮放
- 儲存

狀態提示：

- 目前開啟的 DB 或工作資料夾
- 是否有未儲存變更
- 目前 as-of date
- 是否有資料驗證錯誤

## 4. 左側面板

### 4.1 資料總覽

顯示：

- 公司數量
- 投資關係數量
- 未解析公司數量
- 有缺漏 metadata 的公司數量
- 目前日期有效的節點與連線數量

### 4.2 篩選器

Phase 1 篩選：

- 所在地區
- 使用幣別
- 主營項目
- 是否缺少公司代碼
- 是否缺少公司全稱
- 是否有投資關係

### 4.3 圖層

Phase 1 圖層：

- 公司圖卡
- 投資連線
- 持股比例
- 投資股數
- 資料警示

Phase 2 保留圖層：

- 損益摘要
- 關係人交易線
- 交易金額
- 損益分色

## 5. 中央投資架構圖

### 5.1 公司圖卡

第一版公司圖卡顯示：

```text
公司簡稱
公司代碼
所在地區
使用幣別
主營項目
資料狀態標籤
```

展開或右側面板顯示：

```text
公司全稱
代號
備註
版本歷史
```

資料狀態標籤：

- 缺公司代碼
- 缺公司全稱
- alias 衝突
- 無有效 profile
- 無投資關係

### 5.2 投資連線

投資線方向：

```text
上層公司 -> 下層公司
```

線上顯示：

```text
佔股比
```

hover 或選取後顯示：

```text
投資股數
下層總股數
變更日期
有效期間
來源批次
```

### 5.3 圖面操作

必要操作：

- 拖曳公司卡片
- 點擊公司卡片
- 點擊投資線
- 放大縮小
- 畫布平移
- 搜尋後聚焦公司
- 自動排版
- 顯示某公司上下游
- 切換 as-of date 後保留布局

### 5.4 圖面布局記憶

當使用者拖曳節點，系統應保存節點位置。

切換日期時：

- 同一 company_id 若仍存在，沿用原位置。
- 新出現節點由自動排版補位。
- 消失節點不刪除布局紀錄。
- 使用者可重置布局。

## 6. 右側編輯面板

### 6.1 未選取狀態

顯示目前資料摘要與提示：

- 目前 as-of date
- 有效公司數
- 有效投資線數
- 資料驗證狀態

### 6.2 公司選取狀態

欄位：

- 公司代碼
- 公司簡稱
- 公司全稱
- 公司主營項目
- 代號
- 所在地區
- 使用幣別
- 備註
- 有效起日

輔助區：

- alias 清單
- 版本歷史
- 投資上層
- 投資下層
- 驗證訊息

儲存時必須確認：

```text
本次變更生效日期
```

### 6.3 投資線選取狀態

欄位：

- 變更日期
- 上層公司
- 下層公司
- 投資股數
- 下層總股數
- 佔股比
- 備註
- 有效起日

若投資股數與下層總股數都有值，系統可建議佔股比，但使用者確認前不得靜默覆蓋。

## 7. 底部時間軸

必要控制：

- as-of date 日期選擇器
- 前一個變更日
- 下一個變更日
- 顯示目前日期有效節點/連線數量

顯示模式：

- 單一日期查詢
- 變更日清單

Phase 2 可擴充為 period view。

## 8. 匯入視窗

三步驟：

1. 貼上或選檔。
2. 欄位 mapping。
3. 驗證與套用。

驗證畫面應列出：

- 成功解析列數
- unresolved 公司
- alias 衝突
- 必填欄位缺漏
- 日期格式錯誤
- 百分比格式錯誤
- 會新增的公司
- 會新增或更新的投資關係
