# 產品規格

## 1. 背景

集團財務人員經常需要維護多家公司之間的投資關係、基本資料、公司名稱變動、股權比例變化，以及不同日期的集團架構狀態。這些資料通常分散在 Excel、簡報、手繪架構圖、會計底稿與內部紀錄中。

本工具目標是提供一個純本機、圖像式、時間可回溯的筆記本，讓使用者可以管理公司 metadata 與投資架構，並依指定日期查看當時有效的投資關係。

## 2. 產品目標

- 讓財務人員以公司圖卡和連線方式檢視集團投資架構。
- 讓公司代碼、公司簡稱、公司全稱可以互相查找並指向同一家公司。
- 讓投資架構變更可以按時間保存，不覆蓋歷史。
- 讓 Excel 貼上資料可以快速轉入圖像化架構。
- 讓資料可以完整匯出為本機檔案，避免被工具綁死。
- 為 Phase 2 的收入損益分析與關係人交易圖保留擴充接口。

## 3. 非功能性要求

### 3.1 純本機

- 工具必須可以在無網路狀態下完整使用。
- 不得呼叫任何外部 API。
- 不得自動上傳資料。
- 不得要求使用者登入雲端帳號。
- 所有工作資料應存在使用者指定的本機資料夾或 SQLite DB。

### 3.2 可追溯

- 任何公司 metadata 變更不得直接覆蓋舊資料。
- 任何投資關係變更不得直接覆蓋舊資料。
- 使用者應能查看每筆資料的版本歷史。
- 使用者應能依 as-of date 查詢當時有效狀態。

### 3.3 可匯出

- 必須支援匯出 SQLite DB。
- 必須支援匯出 CSV。
- 必須支援匯出文字 schema 檔。
- 匯出資料重新匯入後，應能重建相同的公司與投資架構。

## 4. 使用者角色

### 4.1 集團財務維護者

負責新增公司、修改 metadata、維護投資股數與持股比例、匯入 Excel 底稿。

### 4.2 財務主管或審閱者

主要查看指定日期的集團架構，確認持股比例、地區分布、公司基本資訊是否正確。

### 4.3 未來擴充角色

Phase 2 後可加入稅務、關係人交易、管理會計或法遵審閱角色。

## 5. Phase 1 核心流程

### 5.1 新增公司

1. 使用者點擊新增公司。
2. 輸入公司代碼、公司簡稱、公司全稱、主營項目、代號、所在地區、使用幣別。
3. 指定有效起日。
4. 系統建立 company identity、alias、profile SCD 紀錄。
5. 圖面新增公司圖卡。

### 5.2 建立投資關係

1. 使用者選擇上層公司。
2. 選擇或新增下層公司。
3. 輸入變更日期、投資股數、下層總股數、佔股比。
4. 系統建立 investment edge SCD 紀錄。
5. 圖面新增上層到下層的投資連線。

### 5.3 切換 as-of date

1. 使用者在底部時間軸選擇日期。
2. 系統查詢該日期有效的公司 metadata。
3. 系統查詢該日期有效的投資架構。
4. 圖面更新節點與連線。
5. 手動布局位置應盡量保留。

### 5.4 Excel 貼上匯入

1. 使用者從 Excel 複製表格。
2. 在匯入視窗貼上。
3. 系統解析表格並猜測欄位 mapping。
4. 使用者確認 mapping。
5. 系統執行公司識別、資料驗證與預覽。
6. 使用者確認套用。
7. 系統寫入本機 DB。

## 6. Phase 2 保留接口

Phase 1 應保留以下擴充點，但不實作完整分析：

- financial metric table。
- company card 的財務摘要區。
- style rule 的損益分色條件。
- future edge type 的關係人交易連線。
- parser rule 對財務科目的欄位 mapping。

Phase 1 可以建立空表或 interface，但不要求輸入或計算損益資料。

## 7. MVP 驗收句

同一份本機資料，在選取 2026-03-31 與 2026-06-30 時，可以正確顯示不同的公司投資架構，且所有修改都不覆蓋歷史資料。
